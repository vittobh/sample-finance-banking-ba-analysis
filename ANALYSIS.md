## 1. Core components & features

Core Banking Solutions encompass: new accounts · loans · interest calc · deposits/withdrawals · customer relationships. The platform enables cross-branch communication, deposit + loan processing.

**Architecture choice — Web app advantages:** simpler dev, fewer requirements, multi-platform support, lower cost.

**Digital signature** — inline with U.S. ESIGN Act 2000 + UETA. High security via layered auth + permissions. SaaS web app feels native + cloud-deploy-friendly for short release cycles.

**Analytics:** BI layer to visualize historical + current data in real time — pattern spotting, bottleneck detection, KPI tracking. SQL + cloud DB integration for speed/scale/reliability.

### Feature list
1. **Sales & Account Opening** — capture first-time visitors via CRM (Salesforce integration)
2. **Digital Signature** — tablet / kiosk e-sign; encrypted signatures, timestamps, PKI, mass-sign, audit
3. **Lead Management** — potential customer capture in CRM
4. **Customer Onboarding** — paperless, few-click digitisation via CRM
5. **Digital Cash Management** — day-to-day transactions, OLTP
6. **Third-Party Integration** — Atom/Molecule UI/UX for CRM, ERP, etc.; ops tracking
7. **Secure Data Management** — customer, business, transactional
8. **Mobile Applications** — anywhere account access; doc upload via device; fast/responsive
9. **Advanced Security** — dual auth with OTP
10. **Account Management** — product engine with variants per account type / line of business
11. **Account Services:**
    - SMS / email notifications for transactions
    - Account settings · Loan service · Investments (bonds/stocks/MF) · Insurance (health/term/personal/property) · Credit (cards/overdraft/flexicash) · Forex
    - Payment options: credit cards, PayPal, offline
    - Customer payment forms or carrier payment-page links
    - Statements & reports

## 4. Scenario — Customer Onboarding

### Use case
| Field | Value |
|---|---|
| Use Case ID | 0001 |
| Name | Customer Onboard in Banking Tablet app by Teller (MVP1) |
| Created by | Vittobha Vignesh S |
| Actors | Bank Teller |
| Description | Teller onboards a new customer via the interactive banking app on tablet. |
| Preconditions | Banking app opened + authenticated with Teller's credentials. POST-1: Teller login Accepted. |
| Postconditions | POST-2: Customer registration Successful. |
| Normal course | Open app in mobile/tablet → log in → navigate to Banking menu → interact with UI to complete registration. |
| Alt courses | None |
| Exceptions | None |
| Includes | Customer digital signature · Loan services option |
| Priority | Business Critical |
| Frequency | Once per visit |
| Business rules | Paperless account opening via digital signature; only registered customers can avail loan service |
| Special requirements | 24/7 access · response times comparable to common apps · available on App Store / Play Store |
| Assumptions | CASA regulations followed · electronic signature for customer/business banking + loans + retail financing |

### User story
**ID:** 0001
> **As a** Bank Teller
> **I want to** register a customer for account opening by filling customer basic details in digital e-forms, selecting the account type from the dropdown in the Tablet banking/finance app, and capturing the customer's e-signature on the tablet, then submit
> **So that** I get a pop-up confirming the customer's banking account is created, displaying the account details.

### Acceptance criteria (Feature-Driven Development)
**Given**
- Teller logged into the Mobile Banking app (Android / iOS tablet).
- Username field per Teller spec.
- Password ≥ 7 chars, 1 number, 1 uppercase, 1 lowercase, 1 special char.

**When**
- Click menu → select **Customer Registration**.
- Add KYC mandatory basic details with proper data types: Name, Address, Contact, Postal Code, Account Type, SSN, Gender, etc.
- Capture **Photo ID** via device camera.
- Capture **e-signature** from customer on summary page; tick terms checkbox.
- Click **Submit**.

**Then**
- Navigate to confirmation window showing customer name, Application Number, status = Open.

### Must / basic attributes
- E-forms, ID capture via camera, e-signatures with digital processes in the banking app (Android / iOS).
- Menu → register new customer → select account type → fill basic + mandatory details per CASA regs.
- **Next** → photo-ID capture + digital sign on tablet → tick terms → **Submit**.
- Confirmation page with Application ID.
- Email + SMS notification on successful account opening.

## Data analysis
- **Customer table** (SQL): N rows, columns include `CustomerID`, `CustomerName`, `ContactName`, `Address`, `City`, `PostalCode`, `SSN`, `Gender`, `Country`.
- Captured **ID + signature images** are typically >1 MB → store in filesystem, not RDBMS. Use a **separate table** for image references.
- **AI automation:** automated CRM + Salesforce Einstein AI + Pivotal CRM + KYC + Lead Management system.
